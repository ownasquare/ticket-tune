# Ollama artifact contract

This directory contains the human-readable form of the Modelfile generated by
`tickettune.export`. For Qwen models, the supported path is deliberately:

1. Reload the exact base model without 4-bit or 8-bit quantization.
2. Apply the PEFT adapter and call `merge_and_unload(safe_merge=True)`.
3. Save merged Hugging Face weights as Safetensors.
4. Convert that merged directory with llama.cpp tag `b9637`, commit
   `aedb2a5e9ca3d4064148bbb919e0ddc0c1b70ab3`.
5. Quantize the F16 GGUF if desired, render a GGUF-only Modelfile, and run
   Ollama creation.

There is intentionally no `ADAPTER` directive in this template. Ollama's
documented direct Safetensors-adapter list does not include Qwen, and its import
guide recommends non-QLoRA adapters even for listed architectures.
